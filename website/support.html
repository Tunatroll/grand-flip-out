Support & troubleshooting

Most issues are either "no API key" or "wrong URL." Here's the usual stuff.

"API status: disabled" or "Waiting for data"

  • Turn on Market API polling in Configuration → Grand Flip Out.
  • In Grand Flip Out API, set the Server URL (e.g. https://grandflipout.com) and paste your API key. Get a key from the dashboard if you don't have one.
  • If you're behind a strict firewall, allow outbound HTTPS—the plugin only talks to your API over HTTPS.

401 or "Invalid API key"

  • Copy the key again from the dashboard—no extra spaces. Keys start with gfo_.
  • If you created a new key, the old one stopped working. Use the new one in the plugin.

Rate limit (429)

  • Too many requests in a short time. Bump the poll interval in config (e.g. 60 or 120 seconds) so the plugin doesn't hit the API as often.

Trades not showing in Trade History

  • Make sure Enable local trade tracking is on in Grand Flip Out config.
  • Trades are recorded when an offer actually completes (buy or sell). Putting in an offer isn't enough—it has to fill.

No opportunities in the Live Market tab

  • Opportunities come from the API. If the server is slow or has no data yet, the list can be empty. Check the API status line in the panel.
  • You can try a different strategy in config (e.g. high_volume or high_margin) if your backend supports it.

Is this allowed?

Yes. Grand Flip Out doesn't automate anything—no clicks, no movement, no input simulation. It just shows you data and records your GE activity. Your trade history and flip logs stay on your PC unless you export them. The plugin is built to stay within RuneLite and Jagex policy.
